## インシデント対応手順

### 1. データ漏洩検知時

```
1. 即座にサービス停止を検討
2. 影響範囲の特定
3. 経営陣への報告
4. 必要に応じて個人情報保護委員会へ報告
5. 該当ユーザーへの通知
6. 原因調査と再発防止策
```

### 2. 不正アクセス検知時

```
1. 該当アカウントの即時無効化
2. セッションの強制終了
3. パスワードリセット強制
4. ログの保全
5. 警察への相談検討
```
